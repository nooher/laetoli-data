Storage

Sovereign object storage: bucket + object metadata live in Postgres (with RLS), the bytes live on a filesystem volume — no MinIO/S3, lighter on a Pi.

Working with files

Get a bucket handle with db.storage.from(bucket), then upload, download, list, or remove. All methods return the { data, error } envelope.

storage.ts 
const bucket = db.storage.from('media');

// Upload bytes (Blob | ArrayBuffer | Uint8Array | string).
await bucket.upload('covers/a.png', file, { contentType: 'image/png' });

// Download an object's bytes as a Blob.
const { data: blob } = await bucket.download('covers/a.png');

// List objects (optional path prefix).
const { data: objects } = await bucket.list('covers/');

// Remove one or more objects.
await bucket.remove(['covers/a.png']);

Public vs private

A public bucket serves objects by URL with no auth — build the link synchronously. A private bucket requires a time-limited signed URL.

urls.ts 
// PUBLIC bucket — synchronous, no network call, no auth.
const { publicUrl } = db.storage.from('public-media').getPublicUrl('logo.png');

// PRIVATE bucket — a signed URL valid for N seconds.
const { data } = await db.storage
  .from('receipts')
  .createSignedUrl('2026/06/r-1.pdf', 3600); // data.signedUrl

Managing buckets

buckets.ts 
await db.storage.createBucket('media', { public: true });
const { data: buckets } = await db.storage.listBuckets();
await db.storage.deleteBucket('media');

Object metadata

Uploads and listings return ObjectMeta rows:

meta.ts 
interface ObjectMeta {
  name: string; bucket: string; path: string;
  size: number; mime: string; owner: string;
  created_at: string; updated_at: string;
}

Writes are owner-scoped and metadata is governed by RLS in the storage schema, so signed-in users only touch their own objects unless a policy says otherwise.
